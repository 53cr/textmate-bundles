TODO list

This is a to-do list for the project. It's where I'll list things that I plan to work on. If you would like to help out, feel free to tackle anything here.

If you have something you would like to add to this list, or you'd really like to see an item bumped up in priority, let me know.

Immediate

  • The "Quit" menu item on modal dialogs is broken, and produces an error message that will mess up your return value if the user selects "CocoaDialog->Quit". Find a way to disable this menu item, or connect its action to something.
  • Timeouts on various controls (especially msgbox and *-msgbox)
  • Figure out why makeFirstResponder:button1 doesn't work in CDTextboxControl (the --focus-textbox handling)
  • Figure out why --with-file doesn't work in CDFileselectControl
  • Better --help handling in code, with one function you can call that gives the user all information. This requires caching the options in CDOptions. Also print out path to the documentation.
  • Better error handling. Since it's called from the command line, stderr should be used to display any problems with the parameters

Eventual

  • New application icon!
  • Create a GNUstep port
  • Resize buttons intelligently to accommodate label size
  • New controls: drop-down menus, checkbox groups, radio groups
  • New control maybe: color selector. Mostly useless though
  • CDOptions has the limitation that a key's value cannot be the same as another key. For example, --title --width will not set the title to "--width". Is this even a problem? I can't imagine it being one for anybody, but you never know.
  • Converge msgbox, inputbox and textbox code. They can just load the correct nib on demand, the code is more or less the same.
